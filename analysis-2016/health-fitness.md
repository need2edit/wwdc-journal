# Health, Fitness & ResearchKit — WWDC 2016 Analysis

**Sessions covered:** 209 (Getting the Most Out of HealthKit), 234 (What's New in ResearchKit), 235 (Building Great Workout Apps), 237 (Getting Started with CareKit), 713 (Health and Fitness with Core Motion), 218 (Keeping Your Watch App Up to Date)

## Headline

watchOS 3 unlocks **background-running workout sessions** — workouts continue in the background for the entire session duration with continuous heart rate and accelerometer streams. iPhone apps can now **start workouts on the watch via `HKHealthStore.startWatchApp(toHandle:completion:)`**. CareKit launches as Apple's open-source patient self-care framework. Wheelchair-distance sample types arrive in HealthKit. Core Motion gains `CMPedometer` extensions and `CMSensorRecorder` for batched recording.

## HKWorkoutSession — the lifecycle (session 235)

```swift
// 1. Authorization
let typesToShare: Set = [HKObjectType.workoutType()]
let typesToRead: Set = [
    HKObjectType.quantityType(forIdentifier: .activeEnergyBurned)!,
    HKObjectType.quantityType(forIdentifier: .distanceWalkingRunning)!,
    HKObjectType.quantityType(forIdentifier: .heartRate)!,
]
healthStore.requestAuthorization(toShare: typesToShare, read: typesToRead) { success, error in }

// 2. Configure
let config = HKWorkoutConfiguration()
config.activityType = .running
config.locationType = .outdoor

// 3. Start
let session = try HKWorkoutSession(configuration: config)
session.delegate = self
healthStore.start(session)

// 4. Open queries for live data
let predicate = HKQuery.predicateForSamples(withStart: Date(), end: nil, options: .strictStartDate)
let devicePredicate = HKQuery.predicateForObjects(from: [HKDevice.local()])
let combined = NSCompoundPredicate(andPredicateWithSubpredicates: [predicate, devicePredicate])

let query = HKAnchoredObjectQuery(type: heartRateType, predicate: combined,
                                   anchor: nil, limit: HKObjectQueryNoLimit) { _, samples, _, _, _ in
    // initial samples
}
query.updateHandler = { _, samples, _, _, _ in
    // streaming heart rate samples — process and update UI
}
healthStore.execute(query)

// 5. Pause / resume
healthStore.pause(session)
healthStore.resumeWorkoutSession(session)

// 6. End
healthStore.end(session)

// 7. Save (in workoutSession(_:didChangeTo:)
let workout = HKWorkout(activityType: config.activityType,
                         start: startDate, end: endDate, workoutEvents: events,
                         totalEnergyBurned: totalEnergy, totalDistance: totalDistance,
                         metadata: [HKMetadataKeyIndoorWorkout: false])
healthStore.save(workout) { _, _ in
    // 8. Add per-sample associations
    let energySample = HKQuantitySample(type: ..., quantity: totalEnergy, start: startDate, end: endDate)
    let distanceSample = HKQuantitySample(type: ..., quantity: totalDistance, start: startDate, end: endDate)
    healthStore.add([energySample, distanceSample], to: workout) { _, _ in }
}
```

## URGENT: Background runtime requires the workout-processing background mode

Add to your **WatchKit extension** Info.plist:

```xml
<key>WKBackgroundModes</key>
<array>
    <string>workout-processing</string>
</array>
```

Without it, the session ends when your app backgrounds. With it, you can:
- Receive continuous heart rate samples (sample-by-sample, not batched).
- Receive continuous accelerometer/gyroscope samples.
- Update the UI for the next time the user raises their wrist.
- Schedule haptic feedback (`WKInterfaceDevice.current().play(.success)` etc.).
- Generate notifications via the new UserNotifications framework.

**URGENT (BEST PRACTICE):** Watch your background CPU usage in Xcode's CPU report tool or Time Profiler. The red dotted line is the system's threshold; exceeding it for too long terminates your app to preserve battery. Apple ships a backtrace log when you cross it.

## NEW workout events (session 235)

`HKWorkoutEventType`:
- `.pause` / `.resume` — generated by HealthKit when you pause/resume the session.
- `.lap` — your code creates these to mark equal-distance partitions (track laps).
- `.marker` — your code creates these for arbitrary points of interest (mile mark, hill summit, "saw a sloth").
- `.motionPaused` / `.motionResumed` (NEW) — HealthKit detects the user stopped/resumed running. **Running activity type only.** Use to ignore samples between these events without manually pausing.

Save events into the workout via the `workoutEvents` parameter when constructing the `HKWorkout`.

## Starting workout from iPhone (NEW in iOS 10)

`HKHealthStore.startWatchApp(toHandle:completion:)` launches the watch app and pre-configures a workout. Useful when the user begins a session on the phone before walking out the door:

```swift
// On iPhone:
let config = HKWorkoutConfiguration()
config.activityType = .running
config.locationType = .outdoor
healthStore.startWatchApp(toHandle: config) { success, error in /* … */ }

// On watch — implement in WKExtensionDelegate:
func handle(_ workoutConfiguration: HKWorkoutConfiguration) {
    let context = workoutConfiguration
    WKExtension.shared().rootInterfaceController?.pushController(
        withName: "Workout", context: context)
}
```

Watch app launches in background, immediately enters workout state. No user interaction on the watch required.

## Activity rings + saved workouts (HIDDEN GEM)

Workouts saved to HealthKit appear in the Activity app's workouts list. **iOS 10 includes phone-saved workouts** (was watch-only) — your iPhone-only fitness app gets credit in Activity now.

For Move ring credit, **add `HKQuantitySample` energy samples** associated with the workout via `add(_:to:)`. Activity sums these to compute Move ring contribution. Make sure the per-sample energies sum to your `totalEnergyBurned` on the workout.

## Wheelchair distance (NEW)

New quantity type `HKQuantityTypeIdentifier.distanceWheelchair`. iOS 10 / watchOS 3 generate these samples for users who self-identify as using a wheelchair (settings: Health > Health Profile > Wheelchair). **For wheelchair-friendly workout apps, query `distanceWheelchair`** instead of `distanceWalkingRunning`.

`pushCount` HKQuantitySample is also new for wheelchair pushes (analogous to step count).

## CareKit (session 237) — open-source patient self-care

Apple's third health-domain framework after HealthKit (data) and ResearchKit (research studies). CareKit is for **ongoing patient care** apps — chronic disease management, post-op recovery, mental health.

Four modules:
- **Care Card** — track adherence to a care plan (medications, exercises, daily activities). User checks off completion.
- **Symptom Tracker** — log symptoms and assessments.
- **Insights** — graph trends in symptoms vs. care plan adherence.
- **Connect** — communicate with care team / family / doctor.

Open source on GitHub. Designed for cross-platform (web equivalents follow). Heavily emphasizes patient ownership of data — local-first, encrypted-at-rest, opt-in-only sharing with care team.

## ResearchKit updates (session 234)

- **New active tasks**: trail-making (cognitive assessment), Stroop test, two-finger swipe test (motor function), audio-based two-tone test (hearing).
- **Charting and dashboard improvements** for studies with longitudinal data.
- **CareKit interoperability** — symptoms logged in CareKit can flow into ResearchKit studies (with patient consent).

## Core Motion improvements (session 713)

- **`CMPedometer.queryPedometerData`** with start/end + completion handler — historical step queries.
- **`CMSensorRecorder.recordAccelerometer(forDuration:)`** — request the system to record accelerometer at 50Hz for a future window (e.g. record sleep). Read back via `CMSensorRecorder.accelerometerData(from:to:)`.
- **`CMAltimeter.startRelativeAltitudeUpdates(to:withHandler:)`** — barometer-based altitude changes. Useful for stairs counted, hill climbs.
- **`CMMotionActivity.cyclingConfidence`** — distinguishes cycling from running/walking. Workouts can auto-classify.

## Best practices summary

- Always set `workout-processing` background mode for workout apps.
- Monitor background CPU; stay below the system threshold.
- Save energy samples associated with workouts → Activity Move ring credit.
- Distinguish wheelchair users via `distanceWheelchair` queries.
- Use `motionPaused`/`motionResumed` events (running) to avoid accumulating data when the user stops.
- For complex apps, support starting workouts both from watch AND from iPhone.
- Maintain workout state when iPhone is unreachable — HealthKit can compute distance without GPS.
- For care plans, evaluate CareKit instead of building from scratch.

## Hidden gems summary

- Background workout sessions stream sample-by-sample heart rate (was batched).
- iPhone apps can launch the watch app in workout state automatically — no user interaction.
- Activity Move ring credits come from associated energy samples, not from the workout's totalEnergyBurned alone.
- Wheelchair distance and push count are first-class quantity types.
- `motionPaused`/`motionResumed` auto-detect running stops without user input.
- CareKit is open source and cross-platform-ready.
